## Patterns UI/UX Communs

### 1. Hero Sections qui Convertissent

#### Structure Optimale

```
+--------------------------------------------------+
|  [Navigation sticky]                              |
+--------------------------------------------------+
|                                                   |
|  HEADLINE PUISSANT (Problem/Solution)             |
|  Sous-titre avec benefice clair                   |
|                                                   |
|  [CTA Primaire]  [CTA Secondaire]                 |
|                                                   |
|  Social Proof: "10,000+ utilisateurs satisfaits"  |
|                                                   |
|  [Visual: Screenshot/Video/Illustration]          |
|                                                   |
+--------------------------------------------------+
```

#### Best Practices 2025

- **60-100% viewport height** sur desktop
- **Headline benefit-driven** vs feature-driven
- **Un seul CTA primaire** visible
- **Load time < 3 secondes** (critique!)
- **Social proof au-dessus du fold**

### 2. Course Cards Efficaces

#### Anatomie d'une Card Performante

```
+----------------------------------+
|  THUMBNAIL (16:9 ou 4:3)         |
|  [Duration badge] [Category tag] |
+----------------------------------+
|  TITRE (2 lignes max, truncate)  |
|  Instructeur/Auteur              |
|  ★★★★☆ 4.5 (1,234 avis)          |
|  [Progress bar si inscrit]       |
|  PRIX ou [CTA Button]            |
+----------------------------------+
```

#### Shadow System

```css
/* 3 niveaux d'elevation */
--shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.05);
--shadow-md: 0 4px 6px rgba(0, 0, 0, 0.1);
--shadow-lg: 0 10px 15px rgba(0, 0, 0, 0.1);

/* Hover state */
.card:hover {
  transform: translateY(-4px);
  box-shadow: var(--shadow-lg);
}
```

### Microinteractions a Implementer

1. **Button press**: Scale down + shadow reduce
2. **Card hover**: Lift up + shadow increase
3. **Success feedback**: Confetti burst + sound
4. **Progress update**: Smooth animation de la barre
5. **Navigation**: Smooth scroll + highlight active
6. **Loading states**: Skeleton screens partout
7. **Error states**: Shake animation + red highlight

### Accessibilite (CRITIQUE pour scolaires)

- **Contraste minimum**: 4.5:1 pour texte normal
- **Focus visible**: Outline clair sur tous elements interactifs
- **Font size minimum**: 16px base
- **Touch targets**: 44x44px minimum
- **Dyslexie-friendly**: Option police OpenDyslexic
- **Dark mode**: Option pour confort visuel

## Conclusion

Pour Kursus, la strategie design recommandee combine:

1. **Confiance** (Coursera, Khan Academy): Palette bleue, design clean
2. **Engagement** (Duolingo): Gamification, mascotte, celebrations
3. **Simplicite** (Brilliant): Focus sur l'apprentissage actif
4. **Social proof** (Domestika, Udemy): Ratings, temoignages, chiffres
